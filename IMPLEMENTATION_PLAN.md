# 참가자 모달 - 신청 통계 기능 추가 구현 계획

## 📋 프로젝트 개요

**목표:** 기존 "참가자" 모달에 신청 통계 기능 추가
- 참가자 신청 일시(`created_at`) 기준으로 통계 분석
- 4단계 시간 단위 상세화: 전체기간 > 주별 > 일별 > 시간별
- PC, 모바일, 앱 완벽 대응 (반응형)

**위치:** 관리자 > 대회관리 > 참가자 탭 > [대회 참가자 클릭] > 모달

**상태:** 📝 **계획 단계 (검토 대기)**

---

## 🎯 요구사항 정리

### 1. 기능 요구사항

#### 1.1 데이터 기준
- **필드:** `registrations.created_at` (참가자 신청 일시)
- **필터:** 취소된 신청 제외 (payment_status !== 'cancelled')
- **범위:** 선택된 대회의 모든 참가자 신청

#### 1.2 시간 단위 (4단계)
```
1. 전체기간
   └─ 모집 기간 전체의 신청 현황 한눈에

2. 주별
   └─ 주단위 신청 분포 (몇 번째 주에 집중?)

3. 일별
   └─ 일단위 신청 분포 (어느 날에 집중?)

4. 시간별 (가장 상세)
   └─ 시간단위 신청 분포 (몇 시에 몰렸나?)
```

#### 1.3 표시 데이터
각 시간 단위별로:
- 📊 신청 수 (당 단위)
- 📈 누적 신청 수
- 📊 확정 수 (당 단위)
- 📈 누적 확정 수

## 📊 시간 단위별 상세 설명

### 3.1 전체기간 탭
- **표시 방식:** 한 줄 (모집 기간 전체)
- **X축:** 시간 구간 (불필요, 텍스트로 표시)
- **Y축:** 신청 수
- **예시:**
  ```
  모집 기간: 2026.02.01 ~ 2026.02.28 (28일)
  총 신청: 250명
  일평균: 8.9명
  ```

### 3.2 주별 탭
- **표시 방식:** 주 단위로 구분
- **X축:** 주차 (1주, 2주, 3주, 4주)
- **Y축:** 신청 수
- **예시:**
  ```
  1주 (2/1-7):   50명 ████████░░ (20%)
  2주 (2/8-14):  70명 ███████████ (28%)
  3주 (2/15-21): 80명 ████████████ (32%)
  4주 (2/22-28): 50명 ████████░░ (20%)
  ```

### 3.3 일별 탭
- **표시 방식:** 일 단위로 구분
- **X축:** 날짜 (02/01, 02/02, ... 02/28)
- **Y축:** 신청 수
- **예시:**
  ```
  02/01: 5명   ██░░
  02/02: 8명   ███░
  02/03: 12명  █████░
  ...
  ```

### 3.4 시간별 탭 (가장 상세)
- **표시 방식:** 시간 단위로 구분
- **X축:** 시간 (00:00, 01:00, ... 23:00)
- **Y축:** 신청 수
- **예시:**
  ```
  00:00-01:00: 2명  ██░░
  01:00-02:00: 1명  █░░░
  ...
  12:00-13:00: 15명 ███████░
  13:00-14:00: 18명 █████████░
  ...
  23:00-24:00: 3명  ███░
  ```

## 🔄 데이터 플로우

```

   ↓

        ↓
4. 해당하는 데이터 필터링 및 그룹화
   ├─ registrations 배열 필터링 (취소 제외)

   └─ 신청 수, 확정 수 계산
        ↓
5. 그래프 데이터 생성
   ├─ chart 배열로 변환
   ├─ 누적값 계산
   └─ AreaChart 또는 BarChart에 전달
        ↓
6. 화면 렌더링
   ├─ 요약 카드 (총신청, 확정, 대기)
   ├─ 시간 단위 버튼
   └─ 그래프
```

```typescript
    pending: registrations.filter(...).length,

```


```typescript

  ]
```

## ⚠️ 주의사항

1. **시간대 처리**
   - `created_at`은 UTC일 수 있음 → KST 변환 필수
   - date-fns와 formatKST 활용

2. **데이터 크기**
   - 참가자가 매우 많으면 시간별 데이터가 많아질 수 있음
   - 필요시 샘플링 고려

3. **누적값 계산**
   - 각 시간 단위별로 누적값 올바르게 계산
   - 취소된 신청은 처음부터 제외

4. **빈 데이터**
   - 신청이 없는 날/시간도 0으로 표시
   - 그래프 끊김 방지

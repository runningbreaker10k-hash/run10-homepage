# RUN10 Flash 기능 기획서

## 1. 개요

### 1.1 배경 및 문제점

**현재 런텐 회원의 사용 패턴**
```
회원가입 → (대회 오픈 시) 대회 신청 → (대회 종료) → ... → (다음 대회까지 방치)
```

| 문제점 | 설명 |
|--------|------|
| 낮은 재방문율 | 대회가 없는 기간에는 사이트 방문 동기 없음 |
| 회원 간 교류 부재 | 같은 회원이어도 서로 만날 기회 없음 |
| 회원 가치 제한 | 회원가입의 혜택이 대회 신청뿐 |

### 1.2 해결 방향

**Flash 기능 추가 후 기대하는 패턴**
```
회원가입 → 대회 신청 + 내 지역 러닝 모임 참여 → 지속적 사이트 방문
```

| 목표 | 기대 효과 |
|------|----------|
| 일상적 방문 유도 | "우리 동네에 오늘 뛸 사람 있나?" 확인 |
| 회원 간 네트워킹 | 같은 지역 러너들과 자연스럽게 교류 |
| 회원 가치 상승 | 대회 + 지역 모임 = 회원가입 이유 증가 |

### 1.3 핵심 컨셉

> **"전국 어디서든, 우리 동네 러닝 모임"**
>
> 대한민국 전체를 "구/군" 단위로 나누어
> 내 관심 지역의 모임을 쉽게 만들고 참여

## 2. 핵심 설계: 구/군 기반 시스템

### 2.1 기본 원리

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   모임 생성자                    참여자                  │
│   ──────────                    ──────                  │
│   [시/도] 선택                   관심 [구/군] 즐겨찾기   │
│      ↓                              ↓                   │
│   [구/군] 선택                   즐겨찾기 지역 모임      │
│      ↓                          우선 표시 + 알림        │
│   상세 장소 입력                                        │
│      ↓                                                  │
│   모임 등록!                                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 행정구역 체계

| 단계 | 예시 | 용도 |
|------|------|------|
| 시/도 | 서울특별시, 대전광역시, 경기도... | 대분류 선택 |
| **구/군** | 서초구, 유성구, 수원시 장안구... | **핵심 필터 단위** |
| 상세 장소 | 반포한강공원 달빛광장 앞 | 자유 텍스트 입력 |

> **"구/군"이 핵심**: 즐겨찾기, 필터, 알림의 기준 단위

### 2.3 장점

| 항목 | 설명 |
|------|------|
| 전국 커버 | 대한민국 모든 지역 대응 가능 |
| 심플함 | 지도/GPS 없이 드롭다운만으로 동작 |
| 명확한 기준 | "구/군" = 누구나 아는 행정구역 |
| 개인화 | 즐겨찾기로 관심 지역만 확인 |
| 확장성 | 알림톡 연동 용이 |

## 3. 사용자 시나리오

### 3.1 모임 만들기 (생성자)

```
김러너(서울 거주)는 오늘 저녁 한강에서 뛰려고 한다.

1. Flash 메뉴 → [모임 만들기] 클릭
2. 지역 선택: [서울특별시] → [서초구]
3. 상세 장소 입력: "반포한강공원 달빛광장 앞"
4. 제목, 시간, 인원 등 입력
5. [등록] → 완료!

→ 서울 서초구를 즐겨찾기한 회원들에게 노출됨
```

### 3.2 즐겨찾기 설정 (참여자)

```
박조깅(대전 거주, 서울 출장 잦음)은 여러 지역에서 뛰고 싶다.

1. Flash 메뉴 → [내 관심 지역] 설정
2. 즐겨찾기 추가:
   - 대전광역시 유성구 (집 근처)
   - 대전광역시 서구 (회사 근처)
   - 서울특별시 강남구 (출장 시)
3. 저장!

→ 이제 이 3개 구의 모임이 우선 표시됨
```

### 3.3 모임 참여하기 (참여자)

```
박조깅이 Flash에 접속한다.

1. 기본 화면: 즐겨찾기 지역(유성구, 서구, 강남구) 모임 표시
2. "갑천 새벽 러닝" (유성구) 발견!
3. 클릭 → 상세 확인 → [참여하기]
4. 완료!

→ 출장 시에는 강남구 모임도 확인 가능
```

### 3.4 전체 지역 탐색

```
이달리기는 여행 중 다른 지역 모임이 궁금하다.

1. [전체 보기] 탭 클릭
2. [부산광역시] → [해운대구] 필터 선택
3. 해운대 러닝 모임들 확인
4. 마음에 드는 모임 참여!
```

## 4. 기능 설계

### 4.1 핵심 기능 (MVP)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **즐겨찾기 지역 설정** | 관심 구/군 복수 선택 | ⭐⭐⭐ |
| **모임 목록** | 즐겨찾기 지역 우선 표시 | ⭐⭐⭐ |
| **모임 만들기** | 시/도 → 구/군 → 상세장소 | ⭐⭐⭐ |
| **모임 상세/참여** | 상세 보기 + 참여/취소 | ⭐⭐⭐ |
| **전체 지역 보기** | 모든 지역 필터 검색 | ⭐⭐ |
| **내 모임** | 내가 만든/참여한 모임 | ⭐⭐ |

### 4.2 입력 항목 설계

**모임 생성 시 입력**

| 항목 | 필수 | 입력 방식 | 예시 |
|------|------|----------|------|
| 제목 | ✅ | 텍스트 (30자) | "한강 야경 러닝" |
| **시/도** | ✅ | 드롭다운 선택 | 서울특별시 |
| **구/군** | ✅ | 드롭다운 선택 | 서초구 |
| **상세 장소** | ✅ | 텍스트 (50자) | "반포한강공원 달빛광장 앞" |
| 날짜 | ✅ | 선택 (오늘/내일/모레) | 오늘 |
| 시간 | ✅ | 선택 (30분 단위) | 19:00 |
| 최대 인원 | ✅ | 선택 (2~20명) | 10명 |
| 소개글 | ❌ | 텍스트 (200자) | "가볍게 뛰실 분~" |
| 예상 거리 | ❌ | 선택 | 10km |
| 페이스 | ❌ | 선택 | 중급 |

### 4.3 모임 상태 관리

```
[모집중] → [마감] → [완료]
   ↓
[취소됨]
```

| 상태 | 조건 | 표시 |
|------|------|------|
| 모집중 (open) | 인원 미달 & 시간 전 | 🟢 모집중 |
| 마감 (closed) | 인원 충족 | 🟡 마감 |
| 완료 (completed) | 모임 시간 경과 | ⚫ 완료 |
| 취소 (cancelled) | 주최자 취소 | 🔴 취소됨 |

### 6.2 SQL 스키마

```sql
-- 행정구역 테이블
CREATE TABLE regions (
  id SERIAL PRIMARY KEY,
  sido VARCHAR(20) NOT NULL,
  sigungu VARCHAR(20) NOT NULL,
  full_name VARCHAR(50) NOT NULL,
  UNIQUE(sido, sigungu)
);

-- 모임 테이블
CREATE TABLE flash_runs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  creator_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  region_id INTEGER NOT NULL REFERENCES regions(id),
  location_detail VARCHAR(100) NOT NULL,
  title VARCHAR(50) NOT NULL,
  run_date DATE NOT NULL,
  run_time TIME NOT NULL,
  max_participants INTEGER NOT NULL DEFAULT 10,
  description TEXT,
  distance VARCHAR(10),
  pace VARCHAR(10),
  status VARCHAR(20) NOT NULL DEFAULT 'open',
  created_at TIMESTAMPTZ DEFAULT NOW(),

  CONSTRAINT valid_status CHECK (status IN ('open', 'closed', 'completed', 'cancelled'))
);

-- 참여자 테이블
CREATE TABLE flash_participants (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  flash_run_id UUID NOT NULL REFERENCES flash_runs(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  joined_at TIMESTAMPTZ DEFAULT NOW(),

  UNIQUE(flash_run_id, user_id)
);

-- 즐겨찾기 지역 테이블
CREATE TABLE user_favorite_regions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  region_id INTEGER NOT NULL REFERENCES regions(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ DEFAULT NOW(),

  UNIQUE(user_id, region_id)
);

-- 인덱스
CREATE INDEX idx_flash_runs_region ON flash_runs(region_id);
CREATE INDEX idx_flash_runs_date ON flash_runs(run_date);
CREATE INDEX idx_flash_runs_status ON flash_runs(status);
CREATE INDEX idx_flash_participants_run ON flash_participants(flash_run_id);
CREATE INDEX idx_flash_participants_user ON flash_participants(user_id);
CREATE INDEX idx_user_favorite_regions_user ON user_favorite_regions(user_id);
```
